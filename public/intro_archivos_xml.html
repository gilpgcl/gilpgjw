Los archivos que se usan con JSF usan la sintaxis de XML. Aún las páginas.

Las reglas principales de XML son:

  • La primera línea de un archivo XML debiera ser el prólogo, o sea: 

    <?xml version='1.0' encoding='UTF-8' ?>
    encoding se refiere al juego de caracteres, que también puede ser windows-1252 o iso-8859-1, etc.

    Para para que las páginas se muestren correctamente en Internet Explorer hay que omitir el prólogo, lo que obliga a que el juego de caracteres sea forzosamente UTF-8. Afortunadamente Netbeans crea los proyectos con esta codificación. Para verificarla o cambiarla, selecciona tu proyecto y luego los menús:

    File > Project Properties

    Aparece un cuadro de diálogo y hasta abajo está un combo con la etiqueta Encoding.

  • El contenido se expresa con elementos como los siguientes:

    <cuate id="cuate1" nombre="Amanda" apellido="Zapata">
  <vip />
  <dirección>Andador de Corredores #34</direccion>
  <!-- comentario -->
</cuate>

    donde:

    <cuate id="cuate1" nombre="Amanda" apellido="Zapata">
    es la etiqueta inicial.
    cuate
    es el tipo de elemento.
    id="cuate1"
    es el identificador del elemento y es único en todo el documento. 
    nombre="Amanda"
    asigna el valor Amanda al atributo nombre de cuate1. 
    apellido="Zapata"
    asigna el valor Zapata al atributo apellido de cuate1.

    El contenido de "cuate1" son los elementos:

    <vip />
    es un elemento que no tiene contenido.
    <dirección>Andador de Corredores #34</direccion>
    es un elemento cuyo contenido es texto.
    <!-- comentario -->
    es un comentario.

  • Los elementos de HTML se ponen en minúscula.
